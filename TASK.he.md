# 🧩 כלי תצורת מפרסמים DeeperDive — מבחן למועמד

## 🎯 הבעיה

**מהנדסי התמיכה של Taboola** צריכים להציג ולערוך תצורות מפרסמים באופן שוטף. הם משתמשים טכניים שיכולים לעבוד עם CSS/JS בסיסי ומבינים JSON, אך יש להם **ידע מוגבל** — עריכה של קבצי JSON גולמיים מובילה לשגיאות תחביר ותסכול.

כרגע, הם מתמודדים עם:
- מציאת קובצי התצורה הנכונים
- הימנעות משגיאות תחביר ב-JSON (פסיקים, סוגרים, מרכאות)
- הבנה של מבנים מקוננים ומערכים
- מעקב אחרי מה השתנה

---

## 👥 המשתמשים שלך

**מהנדסי תמיכה של Taboola:**
- טכניים ובעלי נטייה לנתונים
- נוחים עם לוחות בקרה וכלים
- יכולים ליצור פתרונות CSS/JS פשוטים כשצריך
- מבינים את מבנה ה-JSON אבל יש להם **ידע מוגבל** — הם יודעים מה זה, אך שגיאות תחביר, מבנים מקוננים, ועריכות מורכבות עלולות להכשיל אותם
- צריכים לעבוד במהירות ובביטחון
- לא צריכים לדאוג לשבירת התחביר או מבנה ה-JSON

**המטרה שלך:** עצב ובנה כלי שהופך את עריכת התצורה ל-**בטוחה, ויזואלית ואינטואיטיבית** עבורם. הם מבינים את הנתונים, אבל צריכים הגנה מפני טעויות נפוצות.

## 📊 הנתונים

יש לך נתוני תצורת מפרסמים בפורמט JSON:

- `publishers.json` — רישום של כל המפרסמים  
- `publisher-aurora.json`  
- `publisher-borealis.json`  
- `publisher-cascade.json`  
- `publisher-summit.json`

כל תצורת מפרסם מכילה שדות כגון:
- מידע בסיסי: `publisherId`, `aliasName`, `isActive`
- תצורות עמוד: מערך של עמודים עם `pageType`, `selector`, `position`
- כתובות לוחות בקרה: `publisherDashboard`, `monitorDashboard`, `qaStatusDashboard`
- שדות אופציונליים: `customCss`, `tags`, `allowedDomains`, `notes`, וכו׳.

**הערה:** לתצורות יש שדות מעט שונים — הכלי שלך צריך להתמודד עם זה בצורה גמישה.

## 🛠️ הקשר טכני

**דרישות טכנולוגיות:**
- **TypeScript** — כתוב את הפתרון שלך ב-TypeScript (ללא מסגרות כמו React, Vue, Angular וכו׳)
- HTML, CSS ו-TypeScript פשוטים — בנייה מהיסוד
- ESLint מוגדר ומוכן לשימוש

**זמין:**
- `index.html` בסיסי ב-`/public/`
- קבצי נתונים ב-`/data/`: קבצי JSON של התצורות
- שרת Express ב-`/src/server.ts` (אופציונלי — אם יש לך זמן)
- מהדר TypeScript ו-ESLint מוגדרים

**הפעלת הכלי:**

```bash
npm install
npm run dev    # Runs on http://localhost:3000
```

**שתי גישות:**
- **צד לקוח בלבד:** טען קבצי JSON ישירות באמצעות `fetch('/data/publishers.json')`
- **עם שרת:** השתמש בנתיבי API (`GET /api/publishers`, `GET /api/publisher/:filename`, `PUT /api/publisher/:filename`)

בחר מה שנכון לעיצוב שלך.

**שימוש ב-AI/כלים חיצוניים:**
- אתה יכול להשתמש ב-AI, Cursor או כלים אחרים
- **חשוב:** אתה חייב להבין את כל הקוד בפתרון שלך — נעבור על הזרימה בקוד יחד
